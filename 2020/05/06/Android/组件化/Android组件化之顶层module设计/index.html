Android组件化之顶层module设计

2020-05-06 Android组件化

今天开始了Android组件化顶层module设计的实现

设计原则

  • 设计目的：为了快速的新建APP应用
  • 设计思路：module解耦式分离

设计图

设计细节分析

module命名

module层命名方式

  • 应用层APP（app）命名方式为：app
  • 具体业务层（SpecificBusiness）命名方式为：SBModule + 具体业务名称
  • 基础业务层（BasicBusiness）命名方式为：BBModule + 基础业务名称
  • 基础服务层（BasicService）命名方式为：BSModule + 基础服务名称
  • 公共服务层（BasicCommonService）命名方式为：BSCommon

module 分工

业务层module

  • app：仅仅充当一个APP壳的作用
  • SBModuleCard 卡片业务
  • SBModuleNews 新闻业务
  • SBModuleAlarm 闹钟业务
  • BBModuleSplash 启动页面
  • BBModuleLogin 登录注册
  • BBModuleAbout 关于我们
  • BBModuleSetting 系统设置

服务层module

  • BSModuleNetwork 网络服务
  • BSModuleView 公共view
  • BSModuleSDK 三方SDK封装
  • BSCommon base、application、公用SDK
